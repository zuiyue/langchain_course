# LangChain Router 

## 核心概念

### Router 是什么？

Router 是一种**预处理分发器**，在入口处对输入查询进行分类，并将其定向到处理特定知识域的专门智能体。

```
用户查询 → Router（分类） → 专门代理 A/B/C
                           → 结果合成 → 最终答案
```

### Router vs 其他模式对比

| 特性 | Router | Subagents | Handoffs |
|------|--------|-----------|----------|
| 决策方式 | 单次分类/规则匹配 | 主智能体动态决策 | 状态驱动切换 |
| 上下文记忆 | 无状态（每次独立） | 有状态（主智能体维护） | 有状态（保持上下文） |
| 执行方式 | 并行扇出 | 串行调用 | 串行流转 |
| 适用场景 | 输入类别清晰 | 多领域一次性查询 | 多阶段对话流 |
| 复杂度 | 最轻量 | 中等 | 中等 |

**优势**:
- ✅ 上层代理处理多轮对话和记忆
- ✅ Router 保持简单，每次请求独立
- ✅ 避免 Router 自己管理历史的复杂性

## Router 的核心 API

| API | 用途 | 返回值 |
|-----|------|--------|
| `Command(goto=...)` | 路由到单个目标节点 | `Command` 对象 |
| `Send(node, input)` | 并行扇出到多个节点 | `Send` 对象列表 |

## 最佳实践

1. **优先无状态**: 大多数场景每个请求独立路由即可
2. **有状态用工具封装**: Router 工作流封装为工具，上层代理调用
3. **多轮对话改用 Handoffs**: 如果需要流畅的多轮上下文路由
4. **分类用 LLM**: 使用 `with_structured_output` 做智能分类
5. **并行用 Send**: 需要同时查询多个领域时使用 `Send` API
