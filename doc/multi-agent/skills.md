# LangChain Skills 

## 核心概念

### Skills 是什么？

Skills 是**最轻量级**的多智能体模式。将特定能力封装为可调用的"技能"，通过工具调用实现**渐进式披露**，将专业提示词和领域知识动态注入工作流。

```
用户查询 → Agent 分析意图 → 调用 load_skill → 获得专用提示词 → 按照提示词回答
```

### Skills vs 其他模式对比

| 特性 | Skills | Subagents | Handoffs | Router |
|------|--------|-----------|----------|--------|
| 重量级 | **最轻量**（提示词模块） | 重量级（独立实例） | 中量级（状态切换） | 中量级（分类分发） |
| 上下文 | **单 Agent 保持** | 独立上下文窗口 | 有状态保持 | 无状态 |
| 控制流 | **Agent 自主决定** | Agent 动态协调 | 状态驱动流转 | 预处理分发 |
| 适用场景 | **单 Agent 多专业** | 多领域强隔离 | 多阶段对话流 | 一次性多领域查询 |





## 最佳实践

1. **渐进式披露**: 不要 upfront 加载所有专业知识，按需调用防止上下文膨胀
2. **技能即模块**: 每个技能应该是独立的提示词+上下文，可独立维护
3. **动态工具注册**: 技能加载时同时注册专属工具，实现能力绑定
4. **分层组织**: 大量技能时使用树状结构，便于发现和管理
5. **引用感知**: 大型代码库/文档只保留路径，按需读取避免上下文爆炸
6. **团队解耦**: 不同团队可独立开发、测试和维护各自的技能模块
