# LangChain Handoffs

## 核心概念

### Handoffs 是什么？

Handoffs 是一种**基于状态动态调整智能体行为**的架构。核心机制是：工具调用更新持久化的状态变量，系统读取该变量以切换智能体配置（提示词/工具集）或路由至其他智能体。


### Handoffs vs Subagents

| 特性 | Subagents | Handoffs |
|------|-----------|----------|
| 控制权 | 主智能体始终控制 | 动态转移 |
| 用户交互 | 子智能体不直接与用户对话 | 每个智能体可直接对话 |
| 状态 | 无状态，每次重新加载 | 有状态，上下文保持 |
| 适用场景 | 多领域一次性查询 | 多阶段对话流 |
| 执行方式 | 并行执行 | 串行执行 |


**适用场景**:
- 每个代理本身就是复杂图（含反思/检索等）
- 需要高度定制的子图结构

## 最佳实践

1. **必须包含 ToolMessage**: 交接工具必须返回与 `tool_call_id` 匹配的 `ToolMessage`，否则对话历史会损坏
2. **明确工具语义**: 区分纯路由工具和附带副作用的工具
3. **结束轮次规范**: 交还用户前，确保最后一条消息是 `AIMessage`
4. **优先使用中间件**: 大多数场景使用单代理+中间件更简单
5. **上下文传递**: 切换代理时只传递手递配对消息，避免传递完整历史导致膨胀
